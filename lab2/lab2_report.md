## Цель работы
Научиться настраивать автоматизированные пайплайны для сборки Docker образов, их публикации в registry и автоматического деплоя при изменении кода.

## Ход работы
1. Подготовлено Flask-приложение с Dockerfile.
2. Создан публичный репозиторий на Docker Hub.
3. В GitHub добавлены секреты `DOCKER_USERNAME` и `DOCKER_PASSWORD`.
4. Создан workflow в `.github/workflows/docker-build.yml`.
5. Выполнен коммит и пуш в ветку `lab2`, затем создан Pull Request и слияние с `master`.
6. Проверен запуск Actions и появление образа в Docker Hub.

## Выводы
В ходе работы освоены принципы CI/CD с использованием GitHub Actions. Настроен автоматический процесс сборки и публикации Docker образа при каждом пуше в основную ветку.
